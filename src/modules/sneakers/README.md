# Sneaker Vault

Sneaker Vault is planned for a later phase and must not be implemented during the Cricket Tracker MVP.

For now, this folder exists only to reserve the module boundary for the future predefined tracker app.

Do not add:

- sneaker database tables
- sneaker routes
- sneaker business logic
- sneaker validation
- sneaker AI flows
- a generic tracker/template builder

When product work begins, Sneaker Vault should be built as its own developer-owned module with explicit routes, data model, validation, services, dashboard, and workflows.
